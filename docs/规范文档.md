# 规范文档

## 界面翻译问题

由于浏览器的自动翻译插件（如网页翻译、Chrome自带翻译等）可能会将科学单位错误翻译（例如将 `kg` 翻译为“公斤”，将 `ms` 或 `m/s` 误翻译为“多发性硬化症”），我们在涉及物理单位或公式符号的 UI 元素上，需要加上 `translate="no"` 或者 `.notranslate` 类名，以防止浏览器自动翻译这些文本。

## 动画效果与物理计算模型

### 动画效果实现：
动态演示的核心基于 `requestAnimationFrame` 驱动的循环。在 `App.tsx` 中，我们维护了 `isPlaying` 与 `time` 状态变量，在播放状态下通过提取每帧之间的时间差 `dt` 传入物理引擎步进函数更新画布的所有元素。控制条有“回退（重置）”、“播放”、“暂停”、“停止（重置）”四个动作支持。

### 物理计算引擎方案：
建立于 `src/lib/physics.ts` 的轻量级物理集成工具。
基于数值积分（Euler 积分法）：
   每一帧计算物体受到的合外力 $F=ma$，根据不同类型的元素做差分。
   更新速度：$v_y = v_y - g \cdot dt$
   更新位置：$y = y + v_y \cdot dt$ ; $x = x + v_x \cdot dt$

其中包含了动摩擦力与重力的计算支持：
若物体包含 `friction`，且处在支撑面上($v_y=0$)，当速度大于一定阈值时产生对应运动反方向的加速度 $a = -\text{sign}(v) \cdot \mu \cdot g$。当速度跨越静止点时将自动平滑冻结速度防止抖动误差。

支持完整的 2D 上下叠加：
   基于坐标排序算法计算下层支撑物，支持多层堆叠防止穿模。

### 碰撞检测处理：
- **检测阶段**：通过长宽计算矩形2D AABB边界（针对一维X轴和Y轴的包围盒检测），支持精准宽度和高度。
- **响应阶段（冲量模型）**：
  可通过无选择时的右侧面板调整全局碰撞配置（完全弹性碰撞、完全非弹性碰撞、非完全弹性碰撞）。
  使用含恢复系数 $e$ 的动量守恒分配计算新速度：
  $v_1' = \frac{m_1v_1 + m_2v_2 + m_2e(v_2 - v_1)}{m_1 + m_2}$
  $v_2' = \frac{m_1v_1 + m_2v_2 + m_1e(v_1 - v_2)}{m_1 + m_2}$
